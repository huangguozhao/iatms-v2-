# 测试报告功能用例图 - AI生图Prompt

## 使用说明
将下方Prompt复制到AI绘图工具（如Midjourney、DALL·E、通义万相等），生成与参考图同风格的用例图。**背景必须为白色**，线条与文字为黑色。

---

## 完整版Prompt（推荐）

```
绘制一张软件测试报告管理系统的用例图，严格遵循以下风格与结构：

【画面风格】
- 背景：纯白色。
- 图形：所有用例为白色填充的椭圆，细黑色描边。
- 文字：黑色中文，居中写在椭圆内。
- 线条：黑色实线；从“主功能椭圆”指向“子功能椭圆”的连线带黑色箭头（箭头指向子功能）。
- 中心：画面正中央有一个小的空心白色圆圈（细黑边），无文字，作为枢纽。

【整体布局】
- 中心小圆在正中央，周围分布若干个主功能椭圆（类似星形/放射状），主功能与中心之间用无箭头直线连接。
- 每个主功能椭圆再向外延伸出若干子功能椭圆，用带箭头的线从主功能指向子功能（箭头指向子功能一端）。
- 布局对称、疏密适中，留白清晰。

【内容结构】
1. 执行汇总（主功能）→ 总用例数、通过数、失败数、通过率
2. 失败明细（主功能）→ 失败用例列表、失败原因/错误信息、详细请求响应
3. 趋势分析（主功能）→ 通过率趋势、失败率趋势、项目质量变化
4. 报告导出（主功能）→ HTML导出、PDF导出、Allure报告

所有标签均为中文，风格简洁专业，适合论文插图。
```

## 精简版Prompt

```
画一张测试报告管理系统的用例图。白底，黑色线条和文字。

结构：中心一个空心小圆，周围四个主功能椭圆（执行汇总、失败明细、趋势分析、报告导出），主功能与中心用无箭头直线连接。每个主功能用带箭头的线指向其子功能：
- 执行汇总 → 总用例数、通过数、失败数、通过率
- 失败明细 → 失败用例列表、失败原因/错误信息、详细请求响应
- 趋势分析 → 通过率趋势、失败率趋势、项目质量变化
- 报告导出 → HTML导出、PDF导出、Allure报告

椭圆白底黑边，中文标签，中心发散布局，专业文档风格。
```

## 风格要点（与参考图一致）

| 要素     | 要求                           |
|----------|--------------------------------|
| 背景     | 纯白                           |
| 中心节点 | 小空心圆，无文字               |
| 主功能   | 椭圆，与中心用无箭头直线连接   |
| 子功能   | 椭圆，由主功能用带箭头指向     |
| 箭头方向 | 主功能 → 子功能（箭头在子功能端）|
| 文字     | 黑色中文，椭圆内居中           |
