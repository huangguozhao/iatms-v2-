# 表3.2 系统角色表

| 角色 | 权限或功能 |
|------|------------|
| **系统管理员** | 1. 登录系统；<br>2. 用户管理（创建、修改、删除、启用/禁用用户）；<br>3. 角色管理（创建、修改、删除角色）；<br>4. 权限管理（分配权限）；<br>5. 项目成员管理；<br>6. 访问所有项目资源；<br>7. 系统设置 |
| **普通用户** | 1. 登录注册；<br>2. 个人信息维护（修改密码、个人信息）；<br>3. 通过项目成员身份获得项目访问权限；<br>4. 访问已加入项目的资源 |
| **项目负责人（Owner）** | 1. 项目所有操作权限；<br>2. 管理项目成员；<br>3. 添加/移除成员；<br>4. 分配项目角色；<br>5. 删除项目 |
| **项目管理员（Manager）** | 1. 项目大部分管理权限；<br>2. 管理项目成员；<br>3. 添加/移除成员；<br>4. 分配项目角色；<br>5. 不能删除项目 |
| **开发人员（Developer）** | 1. 创建、编辑、删除模块；<br>2. 创建、编辑、删除接口；<br>3. 创建、编辑、删除测试用例；<br>4. 不能管理项目和成员 |
| **测试人员（Tester）** | 1. 创建、编辑测试用例；<br>2. 执行测试用例；<br>3. 创建、编辑任务；<br>4. 不能删除模块、接口、用例、任务 |
| **查看者（Viewer）** | 1. 只读访问项目资源；<br>2. 查看模块、接口、测试用例；<br>3. 不能执行任何操作 |

## 角色层级关系

```
系统管理员（Admin）
    │
    ├── 不需要加入项目即可访问所有资源
    │
    └── 普通用户（User）
              │
              └── 必须通过项目成员身份获得权限
                        │
                        ├── 项目负责人（Owner）
                        │         │
                        ├── 项目管理员（Manager）
                        │         │
                        ├── 开发人员（Developer）
                        │         │
                        ├── 测试人员（Tester）
                        │         │
                        └── 查看者（Viewer）
```
