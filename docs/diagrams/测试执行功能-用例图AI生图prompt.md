# 测试执行功能用例图 - AI生图Prompt（图示风格）

## 使用说明
将下方「完整版」或「精简版」复制到AI绘图工具（如Midjourney、DALL·E、通义万相等），生成与参考图同风格的用例图。**背景必须为白色**，线条与文字为黑色。

---

## 完整版Prompt（推荐）

```
绘制一张软件测试执行管理系统的用例图，严格遵循以下风格与结构：

【画面风格】
- 背景：纯白色。
- 图形：所有用例为白色填充的椭圆，细黑色描边。
- 文字：黑色中文，居中写在椭圆内。
- 线条：黑色实线；从“主功能椭圆”指向“子功能椭圆”的连线带黑色箭头（箭头指向子功能）。
- 中心：画面正中央有一个小的空心白色圆圈（细黑边），无文字，作为枢纽。

【整体布局】
- 中心小圆在正中央，周围分布若干个主功能椭圆（类似星形/放射状），主功能与中心之间用无箭头的直线连接。
- 每个主功能椭圆再向外延伸出若干子功能椭圆，用带箭头的线从主功能指向子功能（箭头指向子功能一端）。
- 布局对称、疏密适中，留白清晰，整体像参考的“中心发散”用例图。

【内容结构】
1. 单用例执行（主功能）
   - 箭头指向：配置执行环境、参数覆盖、查看执行结果
2. 批量执行（主功能）
   - 箭头指向：选择用例/模块/项目/套件、多线程并发执行、查看进度与状态
3. 执行模式选择（主功能）
   - 箭头指向：同步执行、异步执行
4. 执行历史记录（主功能）
   - 箭头指向：记录执行信息、按条件筛选历史
5. 失败重试机制（主功能）
   - 箭头指向：配置重试次数与间隔、定时任务重试

所有标签均为中文，风格简洁、专业，适合放在论文中。
```

## 精简版Prompt

```
画一张测试执行管理系统的用例图。白底，黑色线条和文字。

结构：中心一个空心小圆，周围五个主功能椭圆（单用例执行、批量执行、执行模式选择、执行历史记录、失败重试机制），主功能与中心用无箭头直线连接。每个主功能用带箭头的线指向其子功能：
- 单用例执行 → 配置执行环境、参数覆盖、查看执行结果
- 批量执行 → 选择用例/模块/项目/套件、多线程并发、查看进度与状态
- 执行模式选择 → 同步执行、异步执行
- 执行历史记录 → 记录执行信息、按条件筛选
- 失败重试机制 → 配置重试与间隔、定时任务重试

椭圆白底黑边，中文标签，中心发散布局，专业文档风格。
```

---

## 风格要点小结（与参考图一致）

| 要素     | 要求                           |
|----------|--------------------------------|
| 背景     | 纯白                           |
| 中心节点 | 小空心圆，无文字               |
| 主功能   | 椭圆，与中心无箭头直线连接     |
| 子功能   | 椭圆，由主功能用**箭头指向**   |
| 箭头方向 | 主功能 → 子功能（箭头在子功能端） |
| 文字     | 黑色中文，椭圆内居中           |
